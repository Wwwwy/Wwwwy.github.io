블로그를 시작하며...

2017-01-25 10:03:00

개발을 하면서 레퍼런스를 참고하기 위해 구글링을 하면 수많은 사람들이 블로그에 정보를 올려놓는걸 보면서 블로그를 해야겠다라는 생각을 2014년도부터 했었다. 그렇게 생각만 지니고 있다가 기업이력서에 블로그를 요구하는걸 보고 본격적으로 시작하겠다는 마음을 먹었다.

첫 블로그 포스팅을 어떤 주제로 할까 고민하다가 생각한게 나처럼 블로그를 시작할려고 마음은 먹었지만 어떻게 할까라는 생각을 지닌 사람들을 위해서 글을 써보자라는 생각을 했다. 막상 처음 블로그를 시작하면 어디서부터 시작해야할지 난감하다. 대형 포털 사이트의 블로그로 하자니 뭔가 개발자로서의 자존심에 스크래치를 내는거 같고 그렇다고 직접 개발하자니 호스팅부터 시작해서 고민되는게 한두개가 아니다. 이렇게 고민하고 있던 찰나에 Github Pages 라는걸 알게되었다.

What is Github Pages ?

Github Pages는 Github 저장소의 내용을 무료로 웹페이지로 서비스해주는 기능이며 일반적인 HTML 콘텐츠를 지원하는 것 외에도 인기있는 정적 사이트 생성기인 Jekyll을 지원한다.

Github Pages를 알게되는 순간 “이거다!”라는 생각이 들었다.

  1. 무료 호스팅
  2. 내가 원하는대로 커스터마이징 가능

내가 원하던 두가지 조건이 충족되는 순간 이용하기로 결정하고 정보수집을 시작했다.

What is Jekyll ?

Jekyll은 다양한 포맷의 원본 텍스트 파일을 템플릿 디렉토리로부터 읽어서, (Markdown 등의) 변환기와 Liquid 렌더러를 통해 가공하여, 당신이 즐겨 사용하는 웹 서버에 곧바로 게시할 수 있는, 완성된 정적 웹사이트를 만들어냅니다. 라고 친절하게 설명되어있다.

처음엔 Jekyll을 이용하여 블로그를 만들려고 했었다. 친절하게 한국어로 번역된 사이트가 있어서 튜토리얼을 따라하며 이것저것 만져보기도 했고 각종 테마들이 공개되어있어 해당 테마를 커스터마이징해보며 작업을 시작했지만, 정보수집 과정에서 Hexo라는 NodeJs 프레임워크를 알아버렸다. 그리고 바로 계획을 수정했다.

What is Hexo ?

Hexo는 블로그 생성을 위한 NodeJs 프레임워크다. 전부터 NodeJs에는 관심이 많았고 실제로 NodeJs를 이용해서 몇가지를 개발해보기도 했다. 그렇기에 Hexo의 존재를 알게되고 정보수집을 해보면서 아래와 같은 생각으로 Hexo 프레임워크를 이용해야겠다고 결정했다.

  1. 메인페이지에 나온 슬로건 “빠르고 간단하고 강력한 블로그 프레임워크” 처럼 간단한 설치 및 실행으로 빠른 결과물을 볼 수 있다.
  2. 좋은 프레임워크의 기본은 많은사람들이 이용해야 한다고 생각하는데 이미 많은 사람들이 이용하고있다.
  3. 문서화가 훌륭히 잘되어있다. ( 더불어 한국어 번역도 훌륭하다 )
  4. Jekyll 테마처럼 많은 사람들이 좋은 테마를 공개해놨다.

이렇게 블로그를 만들면서 점점 욕심(아름다운 디자인)도 생기고 나름 보람도 느껴가면서 즐겁게 만들었던거 같다. 2017년의 목표를 블로그의 오픈과 꾸준한 포스팅으로 잡았고, 첫 단추를 꿰맨만큼 꾸준한 포스팅이라는 나머지 목표 또한 이뤄보고자 한다. 화이팅!!!
